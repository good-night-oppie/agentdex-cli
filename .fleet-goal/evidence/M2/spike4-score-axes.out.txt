### learned

1. **`Benchmark.score()` is free-form `dict[str, float]`.** Keys are bare names (no `+/-`). Any axis set including `{quality, cost_dollar, wall_clock_sec}` is expressible; the contract only requires keys to match stripped objective names.

2. **Direction lives on `objectives`, not in score values.** `SearchConfig.objective_directions()` maps `+name` → maximize, `-name` → minimize. `pareto.dominates()` branches per key. Mixed-direction multi-objective works natively.

3. **arc_agi3’s `-actions` is the template, not a negation hack.** Scores store raw `actions`; `-actions` tells Pareto to minimize. (The aggregate docstring’s “[0,1] normalize” comment is stale — code still returns raw mean actions.)

4. **Negation is optional, not required.** Storing `-cost` + maximize also works, but maximize-on-raw-cost wrongly prefers expensive candidates — the trap the `/tmp` test catches.

```166:177:/home/admin/gh/bene-main/bene/metaharness/harness.py
    def objective_directions(self) -> dict[str, str]:
        """Parse objectives into {name: 'maximize'|'minimize'}."""
        objectives = self.objectives or ["+accuracy", "-context_cost"]
        directions = {}
        for obj in objectives:
            if obj.startswith("-"):
                directions[obj[1:]] = "minimize"
            elif obj.startswith("+"):
                directions[obj[1:]] = "maximize"
            else:
                directions[obj] = "maximize"
        return directions
```
