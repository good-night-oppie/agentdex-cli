### Reasoning path

1. Measured Hub manifest layer sizes (metadata only, no pulls) for the exact compose list in [ServiceNow/webarena-verified `docker-compose.yml`](https://github.com/ServiceNow/webarena-verified/blob/main/docker-compose.yml).
2. Cross-checked docs “optimized size” table ([environments docs](https://raw.githubusercontent.com/ServiceNow/webarena-verified/main/docs/environments/index.md)) — those look like **uncompressed/on-disk**, ~2× registry compressed.
3. SWE-Bench Pro: HF confirms **731** test rows + `dockerhub_tag`; README states one Hub tag per instance under `jefzda/sweap-images`. Sampled 12 tags via registry API; 10-tag mixed-repo dedup was **&lt;1%**, so treat sample disk as **sum**, not shared base.
4. Laptop fourth-adapter criterion = **disk + RAM + hours a user actually pays**. SWE-Bench Pro scales with N; WebArena’s cost is front-loaded and dominated by GitLab + optional wiki/map.

**Counterargument:** If the fourth slot must cover **browser/web agents**, SWE-Bench Pro is the wrong modality. Then use WebArena-Verified’s **offline eval** (HAR replay, no sites) or a **1–2 site subset** (e.g. `shopping_admin` alone ≈ 1.2 GiB compressed / ~2.9 GB on-disk) — not the full compose + wiki/map.
