What Is KAOS?

KAOS is an open-source, local-first multi-agent orchestration framework. Every agent gets an isolated virtual filesystem backed by a single SQLite file. Full audit trail. Checkpoint and restore. Zero cloud dependency.

The headline feature is Meta-Harness: an automated search loop that finds the best prompt strategy for any LLM task by treating prompt engineering as an optimization problem. Define your task. Give it some data. Run kaos mh search. Come back to a ranked Pareto frontier of harnesses.

v0.6.0 adds CORAL — three tiers of co-evolutionary improvements that make the search dramatically smarter, especially when it gets stuck.

The Demo: Code Review Severity Classification

Let's walk through the full Meta-Harness workflow on a real research problem.

Task: Classify a GitHub PR review comment into one of four severity levels: BLOCKER, IMPORTANT, STYLE, PRAISE.

Dataset: 200 labeled PR review comments. Baseline zero-shot accuracy: 48%.

Full recording: benchmark setup → seed eval → 15 iterations → CORAL pivot → winning harness. Nothing skipped.

Step 1: Define the Benchmark

# benchmarks/code_review.py
LABELS = ["BLOCKER", "IMPORTANT", "STYLE", "PRAISE"]

EXAMPLES = [
    {"comment": "SQL query is vulnerable to injection — must fix before merge",
     "label": "BLOCKER"},
    {"comment": "This N+1 query will cause issues at scale",
     "label": "IMPORTANT"},
    {"comment": "Variable name `x` is not descriptive",
     "label": "STYLE"},
    {"comment": "Nice use of early returns here, much cleaner!",
     "label": "PRAISE"},
    # ... 196 more examples
]

def evaluate(harness_fn, examples=EXAMPLES):
    predictions = [harness_fn(e["comment"]) for e in examples]
    correct = sum(p == e["label"] for p, e in zip(predictions, examples))
    return {"accuracy": correct / len(examples)}

That's the full benchmark contract. KAOS needs a callable that takes input and returns a score dict. The search loop handles everything else.

Step 2: Initialize and Launch

kaos init
kaos mh search -b code_review -n 15 -k 3 --background

The --background flag spawns a detached worker process. Parent terminal dies, search continues. Configure CORAL parameters in kaos.yaml:

provider: claude_code        # uses your CC subscription, zero API cost
stagnation_threshold: 4      # CORAL pivot after 4 non-improving iters
consolidation_every: 6       # CORAL skills heartbeat

The CORAL Pivot

When stagnant_iters reaches the threshold, KAOS injects a PIVOT REQUIRED block into the proposer's digest:

╔══════════════════════════════════════════════════════════════════╗ ║ PIVOT REQUIRED — stagnant=4 best=0.74 ║ ║ ║ ║ Exhausted approaches: ║ ║ • Role-playing (engineer/reviewer) — ceiling at 0.74 ║ ║ • Few-shot examples — 1-4 per class, diminishing returns ║ ║ • CoT with contrast — matched best but did not improve ║ ║ ║ ║ Required: propose an orthogonal direction. Ideas: ║ ║ • Two-step classification (blocker? → severity?) ║ ║ • Structured attribute extraction before classifying ║ ║ • Confidence calibration + abstain on ambiguous cases ║ ╚══════════════════════════════════════════════════════════════════╝

The proposer cannot submit another role-playing variant. It must change the fundamental approach. This is CORAL's stagnation-triggered redirection — the difference between a search that circles the same local maximum forever and one that actually explores the space.

The Winning Harness

SYSTEM_PROMPT = '''You are a senior software engineer reviewing a PR.
Classify the comment using a two-step process:

STEP 1 — Extract attributes:
  impact:       high | medium | low
  scope:        blocks-merge | should-fix | nice-to-have | positive
  correctness:  yes (bug/security/logic error) | no

STEP 2 — Apply rules:
  If correctness=yes AND impact=high  →  BLOCKER
  If correctness=yes AND impact str:
    response = llm(SYSTEM_PROMPT, comment)
    return extract_label(response, valid=LABELS)

No embeddings. No retrieval. No fine-tuning. Just a structurally better prompt — the kind of thing an expert might land on after days of iteration. Meta-Harness found it in 15 automated steps.

What's New in v0.6.0: CORAL Three Tiers

Tier 1 — Stagnation + Pivot

After N non-improving iterations, inject PIVOT REQUIRED into the proposer digest. No more incremental tweaks on a stuck frontier.

Tier 2 — Three-Tier Memory

/attempts/ compact summaries · /notes/ iteration observations · /skills/ reusable patterns persisted across searches.

Tier 3 — Co-Evolution

N agents explore the same benchmark in parallel, auto-syncing best harnesses and skills via a shared hub every 2 iterations.

# Launch 3 co-evolving agents
result = mh_spawn_coevolution(benchmark="code_review", n_agents=3)

# Hub auto-syncs every 2 iterations — or sync manually
mh_hub_sync(agent_0_id)   # push your best, pull theirs
CORAL paper result: 3–10× higher improvement rates vs single-agent search. 36% of successful attempts build directly on another agent's discoveries. Kernel engineering record: 1363 → 1103 cycles with 4 agents. Source: arXiv:2604.01658.
